Step 1.1: Verify DEM Quality

üîç Quality Check Procedure

  2. Right-click layer ‚Üí Properties ‚Üí Source tab

‚ö†Ô∏è Common Issue: Mismatched Extents

  1. Analysis ‚Üí Tools ‚Üí "Extract by Rectangle"
  2. Input: DEM_May2024
  3. Rectangle: Manually draw extent that overlaps both surveys
  4. Output: DEM_May2024_Clipped
  5. Repeat for August DEM using SAME rectangle coordinates

2. Seasonal Elevation Change (SEC) Calculation

Step 2.1: Compute DEM of Difference

üßÆ Raster Calculator Workflow

Open Raster Calculator

  1. Analysis tab ‚Üí Tools ‚Üí Search: "Raster Calculator"
  2. In the expression box, build this formula:
"DEM_Aug2024" - "DEM_May2024"
// Double-quotes required for raster names with spaces or special characters
  1. Output raster: C:\PAIS_Thesis\02_Intermediate\SEC_2024.tif
  2. Click Run
  3. Processing time: ~2-5 minutes depending on extent

üí° Interpreting SEC Values

  • Negative values (red): Erosion occurred (elevation decreased)
  • Positive values (blue): Accretion occurred (elevation increased)
  • Values near zero (white/yellow): Stable (minimal change)
  • Typical range: -0.5m to +0.3m for seasonal changes at PAIS

Step 2.2: Calculate Absolute Change

Why absolute change?

For nest safety, we care about magnitude of change (stability), not direction. A nest losing 0.2m or gaining 0.2m are both "unstable" compared to 0.01m change.

// Raster Calculator expression:
Abs("SEC_2024")

// Output: SEC_AbsChange_2024.tif

3. Nest-Scale Stability Analysis

Step 3.1: Import Nest GPS Coordinates

üìç CSV to Point Layer Conversion

Assuming your CSV looks like this:

NestID Latitude Longitude Date
PAIS001 27.1234 -97.4567 2024-05-15
PAIS002 27.1245 -97.4580 2024-05-18

Import Process:

  1. Map ‚Üí Add Data ‚Üí Browse to nest_locations_2024.csv
  2. Right-click CSV in Contents ‚Üí Display XY Data
  3. Settings:
    • X Field: Longitude
    • Y Field: Latitude
    • Z Field: (leave empty)
    • Coordinate System: WGS84 (EPSG:4326)
  4. Click OK ‚Üí Points appear on map
  5. CRITICAL: Right-click point layer ‚Üí Data ‚Üí Export Features
  6. Save as: nest_locations.shp (permanent shapefile)

‚ö†Ô∏è Coordinate System Mismatch

Your nest GPS is in WGS84 (lat/lon), but your DEMs are likely in UTM Zone 14N. You MUST project the nest points to match the DEM:

  1. Analysis ‚Üí Tools ‚Üí "Project"
  2. Input: nest_locations.shp
  3. Output Coordinate System: Same as DEM (UTM Zone 14N NAD83)
  4. Output: nest_locations_UTM.shp
  5. Use this projected version for all subsequent analysis

Step 3.2: Extract SEC Values to Nest Points

Extract Values to Points Tool

  1. Analysis ‚Üí Tools ‚Üí "Extract Values to Points"
  2. Input point features: nest_locations_UTM
  3. Input raster: SEC_2024
  4. Output: nests_with_SEC.shp
  5. Check "Interpolate values at point locations" (uses bilinear interpolation)
  6. Click Run

Result: Each nest point now has a "RASTERVALU" field containing the seasonal elevation change (meters) at that exact location.

Step 3.3: Create 50m Buffers Around Nests

Buffer Tool - Nest-Scale Context

  1. Analysis ‚Üí Tools ‚Üí "Buffer"
  2. Input features: nest_locations_UTM
  3. Distance: 50 Meters (linear unit)
  4. Dissolve Type: None (keep individual buffers)
  5. Output: nest_buffers_50m.shp
  6. Click Run

Why 50m? Literature suggests nesting females assess habitat quality within ~50m radius. Hatchling dispersal also occurs at this scale.

Step 3.4: Calculate Zonal Statistics

üìä Zonal Statistics as Table

Extracting summary statistics within each buffer

  1. Analysis ‚Üí Tools ‚Üí "Zonal Statistics as Table"
  2. Input zone data: nest_buffers_50m.shp
  3. Zone field: FID (or NestID if available)
  4. Input value raster: SEC_AbsChange_2024
  5. Output table: nest_stability_stats.dbf
  6. Statistics type: Check ALL:
    • MEAN - Average absolute change within 50m
    • STD - Variability (higher = more patchy erosion)
    • MIN - Most stable pixel within buffer
    • MAX - Most dynamic pixel within buffer
    • RANGE - MAX - MIN
  7. Click Run

Interpreting Results:

MEAN Value Interpretation Recommendation
< 0.05m Highly Stable ‚úÖ Safe for in-situ incubation
0.05 - 0.10m Moderately Stable ‚ö†Ô∏è Monitor, consider pilot in-situ
0.10 - 0.20m Dynamic üî∂ Risky - recommend ex-situ
> 0.20m Highly Dynamic ‚ùå Relocate eggs to laboratory

4. Creating Management-Ready Maps

Step 4.1: Stability Classification Map

üó∫Ô∏è Reclassify Tool for Categorical Zones

Converting continuous SEC to discrete stability classes

  1. Analysis ‚Üí Tools ‚Üí "Reclassify"
  2. Input raster: SEC_AbsChange_2024
  3. Reclass field: VALUE
  4. Click "Classify" button
  5. Method: Manual ‚Üí Enter these breaks:
    • 0 to 0.05 ‚Üí New value: 1
    • 0.05 to 0.10 ‚Üí New value: 2
    • 0.10 to 0.20 ‚Üí New value: 3
    • 0.20 to 9999 ‚Üí New value: 4
  6. Output raster: Stability_Classes_2024.tif
  7. Click Run

Apply Professional Symbology:

  1. Right-click Stability_Classes_2024 ‚Üí Symbology
  2. Primary symbology: Unique Values
  3. Value Field: VALUE
  4. Click "Add All Values"
  5. Customize colors and labels:
    Value Label Color (RGB)
    1 Highly Stable (<0.05m) Green (56, 168, 0)
    2 Moderately Stable (0.05-0.10m) Yellow (255, 255, 0)
    3 Dynamic (0.10-0.20m) Orange (255, 170, 0)
    4 Highly Dynamic (>0.20m) Red (255, 0, 0)

6. Common Issues and Solutions

‚ùå Error: "Input rasters must have the same extent"

Solution:

  1. Use Extract by Rectangle to clip both DEMs to a common extent (see Step 1.1)
  2. OR use Mosaic to New Raster with "Match to Extent Of: First Of" option

‚ùå Error: "The tool failed to execute"

Common causes:

  • Output file path contains spaces ‚Üí Use underscores instead
  • Output file already exists and is locked ‚Üí Delete or rename existing file
  • Insufficient disk space ‚Üí Free up space or change output location

‚ùå SEC values seem unrealistic (e.g., ¬±10 meters)

Likely causes:

  • DEMs have different vertical datums ‚Üí Ensure both are NAVD88 or same datum
  • One DEM has incorrect Z-units ‚Üí Check if one is in feet vs. meters
  • Point cloud processing error ‚Üí Review raw LAS files for anomalies
