THESIS IDEA 1: Seasonal Stability Mapping for In-Situ Nest Safety

Quantifying May‚ÜíAugust Morphological Changes During Incubation
‚è±Ô∏è 2-4 hours üîß No coding üìä Management-ready

Research Question

"Which beach segments maintain stable morphology from nesting season (May-June) through hatching season (August-September), making them safe for in-situ egg incubation?"

Why This Matters

PAIS currently relocates all Kemp's ridley eggs to laboratories due to erosion, flooding, and vehicle threats. Identifying naturally stable zones enables a transition to in-situ incubation, reducing costs and improving hatchling fitness (literature shows 74-85% in-situ success when erosion is controlled vs. 34-58% in hatcheries).

Complete Workflow

üìÅ Data Preparation (15 minutes)

1
Load your May and August 2024 DEMs
File ‚Üí Add Data ‚Üí Navigate to your processed DEMs
Ensure both have same spatial extent and 0.1m resolution
2
Verify coordinate systems match
Right-click DEM ‚Üí Properties ‚Üí Source ‚Üí Check spatial reference
If different: Use Project Raster tool to align

üßÆ Calculate Seasonal Elevation Change (30 minutes)

3
Open Raster Calculator
Analysis ‚Üí Tools ‚Üí Search "Raster Calculator"
4
Compute DEM of Difference (DoD)
Expression: "DEM_Aug2024" - "DEM_May2024"
Output: SEC_2024 (Seasonal Elevation Change)
Interpretation: Negative = erosion, Positive = accretion
5
Calculate absolute change
Expression: Abs("SEC_2024")
Output: SEC_AbsChange_2024

üéØ Extract Values at Nest Locations (45 minutes)

6
Import nest GPS points
Map ‚Üí Add Data ‚Üí Select nest_locations_2024.csv
Display XY Data: X field = Longitude, Y field = Latitude
Coordinate System: WGS84 ‚Üí Project to match DEM CRS
7
Extract SEC values to points
Analysis ‚Üí Tools ‚Üí "Extract Values to Points"
Input: nest_locations
Raster: SEC_2024
Output: nests_with_SEC
8
Create 50m buffers around each nest
Analysis ‚Üí Tools ‚Üí "Buffer"
Input: nest_locations
Distance: 50 meters
Output: nest_buffers_50m
9
Calculate nest-scale stability
Analysis ‚Üí Tools ‚Üí "Zonal Statistics as Table"
Input: nest_buffers_50m
Value Raster: SEC_AbsChange_2024
Statistics: MEAN, STD, MAX
Output: nest_stability_stats

üó∫Ô∏è Create Stability Classification Map (1 hour)

10
Reclassify SEC into stability zones
Analysis ‚Üí Tools ‚Üí "Reclassify"
Input: SEC_AbsChange_2024
Reclassification:
  • 0 - 0.05m ‚Üí 1 (Highly Stable - Safe for in-situ)
  • 0.05 - 0.10m ‚Üí 2 (Moderately Stable - Monitor)
  • 0.10 - 0.20m ‚Üí 3 (Dynamic - Risky)
  • > 0.20m ‚Üí 4 (Highly Dynamic - Relocate eggs)
Output: Stability_Classes_2024
11
Apply professional symbology
Symbology ‚Üí Unique Values
Color scheme: Green (1) ‚Üí Yellow (2) ‚Üí Orange (3) ‚Üí Red (4)

üìä Statistical Summary (30 minutes)

12
Export attribute table
Right-click nest_stability_stats ‚Üí Data ‚Üí Export Table
Format: CSV for Excel analysis
13
Calculate summary statistics in Excel:
‚Ä¢ How many nests experienced <0.05m change? (Safe zones)
‚Ä¢ What's the mean absolute change across all nests?
‚Ä¢ Which site (A, B, C, D) has the lowest mean change?

üõ†Ô∏è ArcGIS Pro Tools Used

  • Raster Calculator (Spatial Analyst)
  • Extract Values to Points
  • Buffer (Analysis)
  • Zonal Statistics as Table
  • Reclassify (Spatial Analyst)

üìã Expected Outputs

  • Map 1: Seasonal Stability Classification (4 zones: Highly Stable ‚Üí Highly Dynamic)
  • Map 2: Nest locations color-coded by local stability (50m buffer mean change)
  • Table 1: Per-nest statistics (mean, std dev, max change within 50m)
  • Summary: "X% of 2024 nests were in zones with <0.05m seasonal change (safe for in-situ)"

THESIS IDEA 2: Vehicle Access Impact Analysis

Quantifying Erosion Differences Between Protected and Unprotected Zones
‚è±Ô∏è 3-5 hours üîß No coding üìà Statistical testing

Research Question

"Do vehicle-access zones experience significantly higher seasonal erosion rates compared to no-vehicle-access zones at PAIS?"

Why This Matters

Your RA project explicitly classified sites as vehicle-access vs. no-vehicle-access. Literature shows beach driving decreases hatching success by 15-25% and increases false crawls by 2-3x. Quantifying the morphological impact provides evidence for vehicle restriction policies.

Complete Workflow

üó∫Ô∏è Delineate Vehicle Corridors (1 hour)

1
Load MLS intensity raster or orthophoto
Tire tracks visible as linear features in intensity data
2
Manually digitize vehicle corridors
Edit ‚Üí Create ‚Üí Polygon Features
Trace parallel tire tracks (typically 2-3m wide corridors)
3
Create distance raster from vehicle corridors
Analysis ‚Üí Tools ‚Üí "Euclidean Distance"
Input: vehicle_corridors polygon
Output: distance_to_vehicles

üìä Define Comparison Zones (45 minutes)

4
Create site classification polygons
Draw polygons for each site in your project:
‚Ä¢ Site A: Vehicle-Access
‚Ä¢ Site B: No-Vehicle-Access
‚Ä¢ Site C: Vehicle-Access
‚Ä¢ Site D: No-Vehicle-Access
5
Clip SEC_2024 to each site
Analysis ‚Üí Tools ‚Üí "Extract by Mask"
Repeat for Sites A, B, C, D
Outputs: SEC_SiteA, SEC_SiteB, SEC_SiteC, SEC_SiteD

üßÆ Extract Erosion Statistics (1 hour)

6
Calculate summary statistics per site
Analysis ‚Üí Tools ‚Üí "Zonal Statistics as Table"
Zones: site_polygons
Values: SEC_2024
Statistics: MEAN, STD, MIN, MAX
Output: site_erosion_stats
7
Generate random sample points (for t-test)
Data Management ‚Üí "Create Random Points"
Number: 500 per site (2000 total)
Constrain to site polygons
8
Extract SEC values to sample points
Analysis ‚Üí Tools ‚Üí "Extract Multi Values to Points"
Input: random_sample_points
Rasters: SEC_2024, distance_to_vehicles

üìà Statistical Testing in Excel/R (1.5 hours)

9
Export to CSV and load into Excel
Right-click sample points ‚Üí Data ‚Üí Export ‚Üí CSV
10
Separate into two groups:
Group 1: All points from Vehicle-Access sites (A, C)
Group 2: All points from No-Vehicle-Access sites (B, D)
11
Run two-sample t-test in Excel
Data ‚Üí Data Analysis ‚Üí t-Test: Two-Sample Assuming Unequal Variances
Variable 1: SEC values from Vehicle-Access
Variable 2: SEC values from No-Vehicle-Access
Alpha: 0.05
12
Interpret results:
‚Ä¢ If p < 0.05: Significant difference exists
‚Ä¢ Compare means: Which group has higher erosion?
‚Ä¢ Calculate effect size: (Mean1 - Mean2) / Pooled SD

üìã Expected Outputs

  • Map 1: Vehicle corridor delineation with 50m, 100m, 150m buffer zones
  • Map 2: Seasonal erosion (SEC) for vehicle-access vs. no-vehicle sites
  • Table 1: T-test results showing p-value and mean difference
  • Chart: Box plot comparing erosion distributions between site types
  • Finding: "Vehicle-access zones experienced X% higher mean erosion (p=0.XXX)"
‚è±Ô∏è Total Time: 3-5 hours

THESIS IDEA 4: Multi-Year Habitat Persistence Index

Identifying Consistently Stable Zones Across 2024-2026
‚è±Ô∏è 3 hours üîß Cell Statistics üîÑ 3-year analysis

Research Question

"Which beach zones maintain consistent elevation and morphology across all three years (2024, 2025, 2026), indicating long-term habitat reliability for nesting planning?"

Complete Workflow

üìö Stack Multi-Year DEMs (30 minutes)

1
Ensure all 6 DEMs have identical extent and resolution:
May 2024, Aug 2024, May 2025, Aug 2025, May 2026, Aug 2026
2
If extents differ, use Extract by Mask to clip all to common area

üßÆ Calculate Temporal Statistics (1.5 hours)

3
Cell Statistics - Mean Elevation
Analysis ‚Üí Tools ‚Üí "Cell Statistics"
Input: All 6 DEMs
Statistic: MEAN
Output: Mean_Elevation_2024_2026
4
Cell Statistics - Standard Deviation
Same inputs, Statistic: STD
Output: StdDev_Elevation_2024_2026

üó∫Ô∏è Classify Habitat Persistence (1 hour)

6

7

üìã Expected Outputs

  • Map: Habitat Persistence Index (4 classes from Highly Persistent ‚Üí Highly Dynamic)
  • Table: Area (hectares) and percentage of each persistence class
  • Finding: "XX hectares (YY% of beach) are Highly Persistent (CV<10%) - priority zones for in-situ"

THESIS IDEA 5: SLR Threshold Exceedance Mapping

When Does Habitat Fall Below Critical Thresholds?
‚è±Ô∏è 2 hours üåä Future scenarios üìÖ 2045, 2050, 2060

Research Question

"At what SLR increment (+0.4m, +0.8m, +1.34m) does nesting habitat fall below critical elevation thresholds (1.5m above MHW) at each site?"

Complete Workflow

üåä Apply SLR Scenarios (1 hour)

1
Current suitable habitat (baseline)
Raster Calculator: "Elevation_AboveMHW" >= 1.5 & "Elevation_AboveMHW" <= 4.0
Output: Suitable_Current (1 = suitable, 0 = unsuitable)
2
SLR Scenario 1: +0.4m (2045)
Raster Calculator: ("Elevation_AboveMHW" - 0.4) >= 1.5 & ("Elevation_AboveMHW" - 0.4) <= 4.0
Output: Suitable_SLR_04m
3
SLR Scenario 2: +0.8m (2050)
Same formula with -0.8m
Output: Suitable_SLR_08m
4
SLR Scenario 3: +1.34m (2060)
Same formula with -1.34m
Output: Suitable_SLR_134m

üìä Calculate Habitat Loss (1 hour)

5
Summarize area per scenario
For each raster, calculate total area of suitable pixels
Pixel count √ó pixel area (0.01 m¬≤) = total suitable hectares
6
Create threshold exceedance map
Raster Calculator:
("Suitable_Current" * 1) + ("Suitable_SLR_04m" * 2) + ("Suitable_SLR_08m" * 3) + ("Suitable_SLR_134m" * 4)
Interpretation:
‚Ä¢ Value 10 = Lost by 2045
‚Ä¢ Value 13 = Lost by 2050
‚Ä¢ Value 17 = Lost by 2060
‚Ä¢ Value 20 = Persists beyond 2060

üìã Expected Outputs

  • Map: Threshold Exceedance Timeline (when each zone becomes unsuitable)
  • Chart: Area-loss curve showing hectares lost at each SLR increment
  • Table: Per-site habitat loss percentages at 2045, 2050, 2060
  • Finding: "Site A will lose XX% of suitable habitat by 2050 (+0.8m SLR)"
‚è±Ô∏è Total Time: 2 hours
